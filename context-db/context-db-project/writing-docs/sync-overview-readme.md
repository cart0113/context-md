| File                                       | Audience                | Format         | Sync? | Typical size  |
| ------------------------------------------ | ----------------------- | -------------- | ----- | ------------- |
| `README.md`                                | Humans on GitHub        | Plain markdown | yes   | 100–250 lines |
| `docs/src/overview/overview.md`            | Humans on the docs site | Docsify        | yes   | 100–250 lines |
| `context-db/<project>-project/overview.md` | LLM agents              | Frontmatter md | no    | 20–40 lines   |

The README is the canonical full explainer. The docs site overview mirrors it
with docsify-relative links to detail pages. When either changes, update the
other.

## The agent-facing overview is independent

`<project>-project/overview.md` is **not** a duplicate of the README. It is a
short orientation file for an agent about to modify this codebase — the kind of
thing you would tell a new contributor in 30 seconds before they touched
anything.

Target ~20–40 lines. Lead with a single sentence on what the repo ships, then
3–6 bullets covering only what is **non-obvious from the code**:

- API surfaces or single sources of truth (e.g. "the dispatcher is the API
  surface; subcommands are added under `scripts/prompts/`").
- Generated-vs-hand-edited boundaries ("`docs/src/reference/cli.md` is
  auto-generated by `bin/build-cli-reference.py` — never edit by hand").
- Status of in-flight work ("sub-agent dispatch is functional but tuning;
  defaults remain `main-agent`").
- Where tests live and when to run them.

Close with one line pointing at the README and docs for the user-facing story.
Do **not** restate features, install steps, or marketing prose — those belong in
the README. If a section reads like "what this project is," it is in the wrong
file.

## Never symlink — must be separate files

The agent-facing overview requires YAML frontmatter; the README must not have
it. The docs overview uses docsify-relative links; the README uses absolute
URLs. A symlink forces one format on all three.

## Process

After changing the README or docs overview, check the other for contradictions
and update shared sections. The agent-facing overview is independent — leave it
alone unless its own content needs revision.
